HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for the webpage, appearing in the browser's title bar.
  • The title element is required and also very important for SEO.
  • The section element defines sections in a document that contain thematic content.
  • The h1 through h6 elements represent levels of headings.
  • The ul element is for unordered lists (bulleted lists).
  • The ol element is for ordered lists (numbered lists).
  • The li element is for the list items that fall under the ul or ol.

    CSS-SECTION

    • CSS is cascading style sheets which give content style.
    • A margin indicates how much space we want around the outside of an element.
    • A padding indicates how much space we want around the content inside an element.
    • An external CSS has all CSS rules in a separate file which makes design changes easier.
    • CSS Rule format: selector {property:value} {property:value} ...
    • * selector means all elements on the page will have these style properties.
    • Do Not Repeat Yourself (DRY) example: h1,h2,h3 {color:blue}

    GIT-SECTION

    • Git is for version control.
    • Git is run locally from within a command prompt.
    • git status: checks what branch we are currently on.
    • git checkout main: switches to main branch.

    JAVASCRIPT-SECTION

    • Javascript handles the logic and allows users to interact with the website.
